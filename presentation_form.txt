# 연구의 필요성
### 시스템의 필요성
- 필요성 ::
	:: 빅데이터 관련한 얘기가 굉장히 많이 나오고 있고,
	현대에 있어서 굉장히 중요한 요소가 된것 같습니다.
	영재원에서 엑셀을 다루는 법과 C 언어를 배우는 과정에서,
	C 언어를 가지고 엑셀을 구현하면 빅데이터에 대해 더 깊게 공부할 수 있을 것 같아 준비하게 되었습니다.

- 기존 제품이나 연구의 문제점과 차별성 ::
	:: 기존 제품에 대한 차별성보다는 실제로 데이터를 처리하기 위해 어떠한 과정이 필요한지를 C 언어로 구현하면서 확인하며, 
	C 언어로 엑셀과 같은 프로그램을 만들어 보면서, 빅데이터 처리에서 C 언어가 가지는 장점과 단점 또는 한계점이 무엇인지 알아보는 연구 입구니다.

# 연구 목적
### 시스템의 목적

- 연구 목적 :: 
	:: C 언어로 csv 와 같은 데이터 파일을 처리하는데,
	어떤 장단점이 있는지 확인 및 실제로 프로그램이 데이터를 어떡해 읽어와서 처리하는지에 대한 연구

- 제안하는 연구가 갖는 대표적인 특징 ::
	:: 엑셀 프로그램과 같이 데이터 파일을 읽어 들어 사용자의 입력에 따른 각 데이터의 계산 및 처리를 C 언어로 구현함.

- 연구 (수행) 범위 ::
	:: pointer 와 array 를 사용하여 구현할 예정이며, 특정된 csv 파일을 가지고 함.

# 제안하는 *** 시스템
### 시스템 구조도
- 결과물의 주요 아이디어를 구체적으로 표현 ::
	1. 특정된 csv 파일을 읽어들입니다. ( "," 구분된 파일인데, 엑셀 형식의 파일입니다. )
	2. 읽어들인 파일을 array 에 저장 합니다.
	3. array 에 저장하면서 데이터 타입을 C 언어가 처리할 수 있게 바꾸어 줍니다.
	4. 저장된 array 를 화면에 출력하여 엑셀처럼 눈으로 볼수 있게 합니다.
	5. 사용자에게 - 데이터 화면 출력, - 구현된 함수를 이용한 수식 처리, - 프로그램 종료 기능을 선택 할 수 있게 합니다.
	6. "데이터 화면 출력"이 선택한 경우, 현재 array 를 화면에 엑셀처럼 출력합니다.
	7. "구현된 함수를 이용한 수식 처리"를 선택한 경우, 사용자가 선택한 위치를 받아서 엑셀 함수처럼 계산된 결과를 화면에 출력합니다.
	8. "프로그램 종료"를 선택한 경우 프로그램을 저장합니다.
	9. 원하지 않는 입력이 들어올경우 다시 입력하라는 메세지를 출력합니다.
	10. "프로그램 종료"가 선택되지 않으면, 사용자에게 - 데이터 화면 출력, - 구현된 함수를 이용한 수식 처리, - 프로그램 종료 기능을 선택 할 수 있게 합니다.

- 전체적인 구조를 나타내는 그림 ::
	git 에 파일 첨부 완료 photoGraph -> overallStructure.drawio
- 아이디어의 흐름을 표현하는 Flow Chart (흐름도) 작성 ::
	git 에 파일 첨부 완료 photoGraph -> FlowChart.drawio 

- 아이디어의 알고리즘을 슈도코드 또는 자연어로 기술 ::
	git 에 파일 첨부 완료 photoGraph -> psuedoCode.pcse
